## Описание решения

В рамках задания реализован небольшой HTTP API для работы с рецептами.  
Приложение написано на Rust с использованием библиотек `axum`, `tokio` и `serde`.

Реализованные возможности:

- добавление нового рецепта через `POST /recipes`
- получение списка всех рецептов через `GET /recipes`
- фильтрация рецептов по ингредиентам через `GET /recipes?has=...`
- получение случайного рецепта через `GET /recipes/random`
- получение случайного рецепта с учётом ингредиентов через `GET /recipes/random?include=...`

Особенности реализации:

- данные хранятся в памяти приложения
- после перезапуска контейнера список рецептов очищается
- ингредиенты нормализуются для поиска
- при отсутствии подходящих рецептов сервер возвращает сообщение об ошибке

## Структура проекта

- `app/` — исходный код приложения
- `app/requests.sh` — набор тестовых `curl`-запросов
- `app/Dockerfile` — файл сборки Docker-образа
- `docker-compose.yml` — запуск приложения через Docker Compose
- `README.md` — описание решения

## Как запустить

Проект запускается из корня репозитория командой:

```bash
docker compose up --build
```

# Описание API

## Добавление рецептов

**POST /recipes**

Пример запроса:
```bash
curl -X POST http://localhost:3000/recipes \
  -H "Content-Type: application/json" \
  -d '{"title":"omelette","ingredients":["egg","milk"],"steps":"mix and fry"}'
```

## Получение всех рецептов

**GET /recipes**

Пример запроса:
```bash
curl http://localhost:3000/recipes
```

## Фильтрация рецептов по ингредиентам

**GET /recipes?has=egg,milk**

Возвращаются все рецепты, содержащие указанные ингредиенты

Пример запроса:
```bash
curl "http://localhost:3000/recipes?has=egg,milk"
```

## Получение случайного рецепта

**GET /recipes/random**

Пример запроса:
```bash
curl "http://localhost:3000/recipes/random"
```

## Получение случайного рецепта по ингредиентам

**GET /recipes/random?include=egg,oil**

Пример запроса:
```bash
curl "http://localhost:3000/recipes/random?include=tomato"
```

## Проверка работы

Для проверки можно использовать файл requests.sh, в котором содержатся примеры запросов для:
- добавления нескольких рецептов
- получения полного списка
- фильтрации по ингредиентам
- получения случайного рецепта

## Краткий алгоритм работы
- сервер принимает HTTP-запрос
- при POST JSON преобразуется в структуру рецепта
- рецепт сохраняется в общий список в памяти
- при GET список рецептов возвращается полностью или после фильтрации
- при запросе random выбирается один случайный рецепт из подходящих
